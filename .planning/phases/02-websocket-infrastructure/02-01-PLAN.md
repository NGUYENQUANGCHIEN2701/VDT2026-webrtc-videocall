<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → HTTP /ws upgrade | Anonymous TCP upgrade request hits Spring Security HTTP filter chain (already permitAll in Phase 1) |
| browser → STOMP CONNECT frame | Untrusted client supplies Authorization header containing a JWT; this is the actual auth boundary for WebSocket sessions |
| clientInboundChannel → broker | After ChannelInterceptor sets the principal, the broker treats the session as authenticated for the rest of its lifetime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-01 | Spoofing | STOMP CONNECT frame Authorization header | mitigate | JwtChannelInterceptor.preSend rejects CONNECT frames without a Bearer token, with a malformed token, or with an expired token (per AUTH-04); reuses Phase 1 JwtService for validation — single source of truth |
| T-2-OR | Tampering | Interceptor execution order vs. Spring Security's channel interceptors | mitigate | WebSocketAuthInterceptorConfig declared in a SEPARATE @Configuration with @Order(HIGHEST_PRECEDENCE + 99); per Research Pitfall 1 this guarantees the JWT interceptor runs before Spring Security would otherwise process the CONNECT |
| T-2-IL | Information Disclosure | JWT token in logs | mitigate | JwtChannelInterceptor logs only the resolved username AFTER successful validation; raw token value is never logged or returned in an error message (acceptance criterion 9 grep-gates this) |
| T-2-DC | Denial of Service | Repeated CONNECT attempts with invalid tokens | accept | LAN demo scope; no rate limiting in MVP (per Research Security Domain table — "presence flood acceptable for LAN demo") |
| T-2-CD | Repudiation | Lost audit trail of who connected when | accept | Logging at INFO level captures session-connected username; no separate audit log table required for MVP |
| T-2-SC | Tampering | npm/Maven supply chain — spring-boot-starter-websocket | accept | Official Spring Boot artifact, BOM-managed, no version override, Package Legitimacy Audit in 02-RESEARCH.md disposition "Approved" — no human checkpoint required for this single managed dependency |
</threat_model>
