## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser ↔ STOMP backend `/app/signal` | All signal traffic. Backend overwrites `from` field with authenticated principal (T-2-02 mitigation already in place from Phase 2). |
| Signal handler ↔ RTCPeerConnection | Untrusted `payload` strings passed to `setRemoteDescription` and `new RTCIceCandidate()`. Browser validates; malformed input throws DOMException. |
| User input ↔ CallContext | `startCall(targetUsername)` parameter could be self or empty/invalid username. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-01 | Spoofing | Signal `from` field | mitigate (server-side) | Backend `SignalController` overwrites `from` with `principal.getName()` — already in place from Phase 2. Client treats `msg.from` as trusted authenticated identity. |
| T-4-02 | Tampering | SDP injection via `payload` | mitigate | Wrap entire `handleSignal` dispatch in `try/catch`; on exception call `teardown()` and add `'Connection lost'` toast. Browser's `setRemoteDescription` throws DOMException on malformed SDP — caught by the wrapper. |
| T-4-03 | Spoofing | Self-call via `startCall(authUsername)` | mitigate | `startCall` early returns when `targetUsername === username` from `useAuth()` (defense-in-depth — UserListPage already filters self). |
| T-4-04 | Information Disclosure | ICE host candidates reveal LAN IPs | accept | LAN demo scope per project constraints; STUN-only ICE config. Documented in RESEARCH §Security Domain. |
| T-4-05 | Denial of Service | Unhandled async exception in signal handler | mitigate | Try/catch wrapper with teardown fallback (Open Question 2 from RESEARCH). |
| T-4-SC | Tampering | npm installs | mitigate | No new packages installed in this plan (RESEARCH §Package Legitimacy Audit confirms). |
