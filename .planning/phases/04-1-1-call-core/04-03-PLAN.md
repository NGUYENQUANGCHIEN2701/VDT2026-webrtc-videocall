Implement the full `IncomingCallModal` overlay and the `useRingtone` Web Audio API hook. Delivers the vertical slice: incoming `call-request` signal sets `callStatus = 'ringing'` in CallContext → IncomingCallModal becomes visible → ringtone plays → user clicks Accept or Reject → state transitions. After this plan, a user (callee) sees and hears an incoming call notification and can accept or reject it (CALL-02 + CALL-03 end-to-end).

Purpose: User-visible callee-side capability. Without this plan, the callee has no UI to acknowledge incoming calls.
Output: Modal + ringtone working; pressing Accept hands control off to CallContext (which navigates to /call — handled in Plan 04); pressing Reject sends call-decline and clears state.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `peerUsername` displayed in DOM | Value comes from `msg.from` which is server-overwritten with authenticated principal (T-2-02 mitigation from Phase 2). Trusted source — safe to render. |
| Web Audio API ↔ browser autoplay policy | AudioContext may be created in suspended state. `resume()` is called explicitly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-03-01 | Information Disclosure | Caller username rendered in modal | accept | `peerUsername` is the authenticated caller's identity; legitimate to display. Source is server-overwritten `from` field. |
| T-4-03-02 | Denial of Service | Ringtone never stops (audioCtx leak) | mitigate | Cleanup function in `useRingtone` closes AudioContext on unmount; `stopped` flag prevents scheduled beep from firing after unmount. |
| T-4-03-03 | Cross-Site Scripting | `peerUsername` rendered inside JSX text node | accept | React escapes JSX text by default; no `dangerouslySetInnerHTML` used. |
| T-4-SC | Tampering | npm installs | mitigate | No new packages installed (Phase 4 uses only browser-native APIs and existing dependencies per RESEARCH §Package Legitimacy Audit). |
</threat_model>
