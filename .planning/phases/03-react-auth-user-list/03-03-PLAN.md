<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → STOMP /topic/presence broadcast | Untrusted server payload deserialized into onlineUsers state |
| Browser → /api/auth/logout | Authenticated request invalidates the session server-side |
| Browser localStorage → DOM render | Token-derived username is rendered in the header |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-3-13 | Tampering | Presence payload injection | LOW | mitigate | WebSocketContext (Plan 01) does `JSON.parse(frame.body) as { onlineUsers: string[] }`; UserListPage uses values only as React text (escaped). No HTML injection surface — array of strings rendered via `{user}` and `aria-label=`Call ${user}`` which React text-escapes. |
| T-3-14 | Spoofing | Stale STOMP subscription after logout | LOW | mitigate | Logout sequence calls `disconnect()` (which calls `client.deactivate()`) BEFORE clearing localStorage and navigating. Backend marks user OFFLINE on disconnect (Phase 2 PresenceService). |
| T-3-15 | Information Disclosure | Username from other users in DOM | LOW | accept | Usernames are deliberately broadcast as part of the presence feature (PRES-01). No PII beyond username per phase scope. |
| T-3-16 | Information Disclosure | Logout fails silently if backend returns 401 | LOW | mitigate | Try/catch around `api.post('/api/auth/logout')` (per D-07 documented behavior); STOMP disconnect + localStorage clear + navigate always run. The session is effectively terminated on the client; backend will mark OFFLINE on STOMP disconnect even without the explicit logout call. |
| T-3-17 | Denial of Service | Self-click on own Call button (defense in depth — self is filtered) | LOW | mitigate | RESEARCH Pitfall 4 — self is filtered from `otherUsers` before rendering; the Call button only renders for OTHER users. The button's onClick is a no-op in Phase 3. |
| T-3-18 | Elevation of Privilege | Manually setting a forged JWT in localStorage | LOW | mitigate | ProtectedRoute is a UX guard only — backend independently validates JWT signature on every REST call AND on STOMP CONNECT (Phase 2 JwtChannelInterceptor). Forged token causes STOMP CONNECT to fail; user sees a stuck loading state but cannot read others' presence. Documented in verification step 19. |

**HIGH severity items:** none. Largest residual is T-3-13 (payload injection from a compromised backend), mitigated by React's text-escaping for all rendered values.

</threat_model>
