<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client HTTP POST → AuthController | All three auth endpoints accept JSON from untrusted clients; @Valid annotations + GlobalExceptionHandler form the input-validation perimeter |
| Plaintext password → BCryptPasswordEncoder.encode | Password leaves request body and is hashed before persistence; raw value never written to DB or logs |
| AuthenticationManager.authenticate(username, password) → DaoAuthenticationProvider → CustomUserDetailsService | Credential verification path; failure throws BadCredentialsException → 401 via GlobalExceptionHandler |
| Bearer token → SecurityContextHolder principal → /api/users/me | JWT-derived identity propagated into controller via Authentication parameter |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-16 | Information Disclosure | Password stored in plaintext | mitigate | AuthService.register calls `passwordEncoder.encode(request.password())` before User.builder; raw password never assigned to User.passwordHash. ASVS V2.4. RESEARCH.md Security Domain row 3. |
| T-01-17 | Tampering / Spoofing | SQL injection via username on registration | mitigate | UserRepository.existsByUsername / findByUsername / save use Spring Data JPA parameterized queries. No string concatenation in AuthService or UserController. ASVS V5.3. |
