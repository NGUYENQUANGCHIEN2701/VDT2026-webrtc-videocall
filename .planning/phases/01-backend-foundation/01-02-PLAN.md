    - public Claims extractAllClaims(String token): use `Jwts.parser().verifyWith(getSigningKey()).build().parseSignedClaims(token).getPayload()` — uses NEW 0.12.x API: `.parser()` not `.parserBuilder()`, `.verifyWith()` not `.setSigningKey()`, `.parseSignedClaims()` not `.parseClaimsJws()`, `.getPayload()` not `.getBody()`

    - public boolean isTokenExpired(String token): return `extractAllClaims(token).getExpiration().before(new Date())`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted HTTP request → JwtAuthenticationFilter | Every request crosses here; Bearer token must validate before SecurityContextHolder is populated |
| JWT claims → SecurityContextHolder | Spoofing-critical boundary; JJWT signature verification gates this |
| Password input → BCryptPasswordEncoder | Bcrypt salt + work factor 10 mitigates rainbow/dictionary attacks |
| CORS preflight (OPTIONS) → SecurityFilterChain | CorsConfigurationSource bean must process before auth filter to allow browser preflight |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | Spoofing | Forged JWT (weak secret) | mitigate | `Keys.hmacShaKeyFor()` enforces ≥256-bit key; WeakKeyException thrown on short secret. application.yml default is ≥32 chars. ASVS V6.2. RESEARCH.md Security Domain row 1. |
| T-01-09 | Elevation of Privilege | JWT with no expiry / overlong expiry | mitigate | `.expiration(new Date(System.currentTimeMillis() + jwtExpirationMs))` mandatory in JwtService.generateToken; 24h cap per D-06. ASVS V3.3. RESEARCH.md Security Domain row 2. |
| T-01-10 | Tampering | CSRF on state-changing endpoints | mitigate | `.csrf(csrf -> csrf.disable())` in SecurityFilterChain. Stateless JWT has no session cookie to exploit. ASVS V13.2. Grep gate enforces presence. RESEARCH.md Pitfall 4. |
| T-01-11 | Spoofing | Missing Authorization header | mitigate | JwtAuthenticationFilter skips cleanly when header is absent; `.anyRequest().authenticated()` causes Spring Security to return 401 for protected endpoints. ASVS V4.1. RESEARCH.md Security Domain row 6. |
| T-01-12 | Information Disclosure | Stack trace leakage on uncaught exception | mitigate | GlobalExceptionHandler @ExceptionHandler(Exception.class) returns sanitized `{ "error": "INTERNAL_ERROR", "message": "An unexpected error occurred" }` instead of Spring's default whitelabel error page. ASVS V7.4. |
| T-01-13 | Tampering | SQL injection via username in UserDetailsService | mitigate | CustomUserDetailsService uses `userRepository.findByUsername(username)` which is Spring Data JPA-generated parameterized SQL. No string concatenation. ASVS V5.3. RESEARCH.md Security Domain row 4. |
| T-01-14 | Spoofing | JWT replay after key rotation | accept | Out of Phase 1 scope. No refresh token / blacklist per D-04 (LAN demo). Documented in Deferred Ideas (CONTEXT.md). Revisit if deployed publicly. |
| T-01-15 | Elevation of Privilege | WebSocket bypass of HTTP auth | mitigate | `.requestMatchers("/ws/**").permitAll()` pre-permits the STOMP upgrade path so Phase 2's ChannelInterceptor handles JWT inside the STOMP CONNECT frame (correct auth point per RESEARCH.md Pitfall 7). Without this Phase 2 would fail. ASVS V13.5. |
</threat_model>
