<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker container → host filesystem | Multi-stage Dockerfile runs `java -jar` as non-root `appuser` to limit container escape impact |
| Spring Boot ↔ PostgreSQL JDBC | Datasource credentials sourced from environment (`DB_USER`, `DB_PASS`) with localhost defaults; no secrets in image |
| Flyway migration scripts ↔ runtime | Schema changes are versioned SQL files committed to source — auditable, reproducible |
| JWT secret config ↔ environment | `${JWT_SECRET:default-...}` pattern; default is dev-only marker, real value injected via env var (D-07) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
