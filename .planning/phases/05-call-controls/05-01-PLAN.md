<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser WebRTC API → CallContext | `iceConnectionState` value read from RTCPeerConnection event; value is spec-constrained enum |
| CallContext → CallPage | Toggle state exposed as read-only values; no untrusted input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | toggleMute / UI mute indicator | accept | `track.enabled = false` is enforced in the browser's RTP sender layer; JS-level muting cannot be bypassed at the JS layer without modifying the browser. Acceptable for LAN demo. |
| T-05-02 | Tampering | `iceState` from `pc.iceConnectionState` | accept | Value comes from the browser's WebRTC implementation, not from network input. RTCIceConnectionState values are a browser-controlled enum. No user-controlled input reaches this field. |
| T-05-03 | Information Disclosure | CallContext state (isMuted, isCameraOff) | accept | These are local UI state values; no PII. Visible only within the same browser tab via React context. |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in Phase 5. All capabilities come from already-audited Phase 3/4 dependencies. No slopcheck required. |
</threat_model>
